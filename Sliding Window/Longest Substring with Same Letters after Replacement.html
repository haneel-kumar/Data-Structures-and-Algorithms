Key Insights

The Core Formula

At any window, we want to keep the most-frequent letter and replace everything else. So: replacements needed = windowSize − maxRepeatLetterCount. If this exceeds K, shrink the window by one from the left.

Why maxRepeat Never Decreases 🔒

When shrinking, we don't recompute maxRepeatLetterCount. Why? Because we only care about growing the answer. If no future window reaches the same maxRepeat, the window will just slide forward at constant size — never finding a longer valid window anyway.

No Inner While Loop

Unlike previous dynamic-window problems, shrinking is always exactly one step — decrement the start char's frequency and move windowStart forward. The window never shrinks below its current size; it either grows or slides.

O(26) = O(1) Space

Only lowercase letters are allowed, so the charFrequency map has at most 26 keys — a fixed constant regardless of input size, making space effectively O(1).
