What Makes This Different

Fixed-size window (prev problems)

Window size is always K. Slide forward exactly one step at a time — one element in, one out. Simple and predictable.

Variable-size window (this problem)

Window size flexes. Expand by moving windowEnd right until sum ≥ S. Then shrink from windowStart while sum ≥ S, recording the smallest valid window each time.

Why O(N) and not O(N²)? Even though there's a while loop inside the for loop, each element is added to the window once (outer loop) and removed from the window at most once (inner while). So the total number of operations across the entire run is at most 2N → O(N + N) = O(N).
