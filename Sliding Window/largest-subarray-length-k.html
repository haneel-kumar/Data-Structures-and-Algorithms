Complexity Comparison
O(N × K) time · O(1) space

Brute Force

For every starting index i, loops through the next K elements to compute the sum — re-scanning the overlapping region every time.

O(N) time · O(1) space

Sliding Window

Reuses the previous window's sum. Each step is just one subtraction + one addition — constant work regardless of K.

Key insight: Two adjacent windows of size K share K − 1 elements. The sliding window avoids recomputing those shared elements by keeping a running sum, subtracting the element leaving and adding the element entering — reducing the total work from O(N·K) to O(N).
