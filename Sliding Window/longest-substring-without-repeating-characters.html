Previous problems — frequency map

Stored { char: count }. When the window became invalid, used a while loop to shrink one step at a time until valid again. O(N) total but multiple iterations per repeat.

This problem — index map ⚡

Stores { char: lastSeenIndex }. When a repeat appears, jumps windowStart directly to lastIndex + 1 in one step — no inner loop needed. The Math.max guard prevents moving backward.

Why This Approach Is Different
Index map vs frequency map. Previous problems needed a frequency count to know when to remove a character from the window. Here, because we want zero repeats, we don't need to count — we just need to know where we last saw each character. That lets us skip past the entire stale occurrence in O(1) instead of walking back one step at a time.
The Math.max guard. When a repeated character's previous index is behind our current windowStart (already outside the window), jumping to prevIdx + 1 would move windowStart backward — making the window larger and invalid. Math.max(windowStart, prevIdx + 1) ensures we never shrink backward.
O(N) time — single pass, O(1) jump per repeat
O(K) space — K distinct chars in map
