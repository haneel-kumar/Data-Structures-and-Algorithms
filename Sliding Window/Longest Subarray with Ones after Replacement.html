Differences from Previous Problem

Longest Substring after Replacement

Characters: any letter (a–z). Need HashMap to count frequencies. maxRepeatLetterCount is a high-water mark — never decremented during shrink, even if the dominant char leaves. Window size never shrinks below its max.

This problem (binary)

Characters: only 0 and 1. No HashMap — just increment/decrement a counter. maxOnesCount is decremented when a 1 leaves the window, because counting 1s directly is always accurate. Window can shrink by 1.

Why no HashMap? When input is binary, counting 1s directly is as informative as a full frequency map. We always want to maximise 1s, so there's no need to track which character to keep — it's always 1. Zeros in the window = windowSize − maxOnesCount, which is exactly the number of replacements needed.
O(N) time — single pass
O(1) space — no map needed
