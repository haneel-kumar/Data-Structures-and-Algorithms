Time Complexity Comparison
Brute Force — O(N × K)
Sliding Window — O(N)

In the brute-force approach every new subarray recomputes its sum from scratch, touching K elements each time. The sliding window avoids this by reusing the previous sum — it simply subtracts the element leaving the window and adds the element entering, keeping the work per step constant O(1).

KEY INSIGHT

Two consecutive windows of size K share K – 1 elements. Instead of re-summing those shared elements, the sliding window carries the sum forward, making each step O(1) regardless of K.
